What is HTML and Why Do We Care


What is HTML?

HTML, standing for HyperText Markup Language, is a type of markup language.

A markup language is a system for annotating a document in a way that is syntactically distinguishable from the text. ---- From Wikipedia

In another words, a markup language is essentially a text script with parts that are used to indicate forms, styles or transformation of normal text parts, instead of simply functioning as normal texts.

If we look at an HTML file, we see that some parts of the script is just normal text but there are also parts such as all the tags that are surrounded by angle brackets.

Is there other type of Markup Languages?

Oh, definitely! There are famous others like XML(Extensible Markup Language), XHTML(Extensible HyperText Markup Language), and one very important one that you will or you already have run into, the TeX family (most famous one LaTex).

So why do we care about HTML

A simple answer would be: because this is the core language of web development.

The purpose of a web browser (Chrome, IE, Firefox, Safari) is to read HTML(sometimes XHTML) documents and interpret them into nice(or shaddy) webpages.

Blocks and Layouts


About Blocks

Blocks, tag <div>, is the core of doing webpage layout.

A block, literaly, is a block of space you get on the page. You can image it as a box or tray where you can put everything you want in and you can position it anywhere you think fit without affecting its contents or the contents of other blocks.

One example you've already seen is when we list out three different kinds of lists. Each type of list is put in a block and the outline is shown for you to see what the blocks look like.

Making use of styling for blocks, as we did for the three, we can align blocks horizontally and responsive (If you squeeze the browser horizontally you can see the blocks scooch down under the previous one instead of staying where they are and go out of page scope.)

Responsive design is vital in today's web design for the various platform with different screen/browser window sizes (e.g. laptop compared to mobile devices).

Make the width certain percentage of the browser width so that the block stretch/shrink with the browser would be another way to do it.

An empty block that spanning the whole width of browser would make a nice separation between web contents sometimes especially when stylings have effects on each other. (Do you know there's an empty block right above me? :P)

About Semantic Elements

Elements that tell nothing about their contents are called "non-semantic" elements like <div> while those which do are called "semantic" elements like <form>. HTML5 provides even more semantic elements for different uses that come in handy.

Take a note though that the biggest benefit new semantic elements like header, footer give is for different webs to have same skeleton and naming scheme so data could be better shared and reused. Also it's easier for search engines to id the correct content.
